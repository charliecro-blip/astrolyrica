# Copyright and Source Rules

AstroLyrica stores original symbolic notes, not copied prose. Every database entry should be written in fresh language and should preserve interpretive usefulness rather than source phrasing.

## Public-domain sources

Public-domain sources may be used for concepts, doctrine, transformed imagery, and direct quotation when the source is clearly public domain. If a direct quotation is stored or used, label it as a quotation and record the source in `data/source_notes.yaml`.

## Modern copyrighted sources

Modern copyrighted sources may be used only for concept extraction, non-verbatim notes, bibliography, and comparison. They may help identify interpretive themes, historical disagreements, or terminology, but their expressive language must not be copied into the database.

Do not store:

- Distinctive modern phrasing.
- Memorable sentences.
- Long excerpts.
- Authorial imitation.
- Close paraphrases.

## Charlie-owned writing

Charlie-owned writing can be used for house style and voice calibration. It may inform tone, cadence, taste, and preferred imagery, but the project should still keep reusable entries modular and avoid overfitting every output to one passage.

## Database entry standard

Database entries should store symbolic meaning, images, textures, verbs, and interpretive logic, not copied prose. Prefer compact fields that can be recombined by prompts: keywords, image banks, body language, shadow language, medicine language, sentence patterns, and avoid lists.
